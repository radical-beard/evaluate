# EvaLuate downloads

Drop-in resources for **consuming** EvaLuate — copy what you need into your own project.
Everything here is either generated by EvaLuate itself or runnable example material; none of
it is hand-maintained API documentation.

```
downloads/
  spec/                       # the generated Lua API spec (auto-tracks Godot + EvaLuate)
    evaluate-api.json         #   full machine-readable surface
    evaluate-api.md           #   human/LLM-readable reference
    luacats/                  #   lua-language-server (---@meta) definitions for IDE autocomplete
      godot-core.lua          #     common engine classes (default)
      godot-full.lua          #     every engine class (opt-in; large)
      std.lua                 #     std.* types
      evt-apis.lua            #     capability apis + hook/frontmatter reference
  skill/
    evaluate-scripting/       # an agent skill: teaches an LLM to write EvaLuate scripts
      SKILL.md
      reference/ examples/ common-mistakes.md
```

## Use it

**IDE autocomplete (any editor with the Lua Language Server):** add `spec/luacats` to the
server's library, e.g. in `.luarc.json`:

```json
{ "workspace.library": ["downloads/spec/luacats"] }
```

You then get completion + hover over the whole `godot.*` and `std.*` surface — typed
methods, properties, signals, enums.

**Agent / LLM setup:** copy `skill/evaluate-scripting/` into your project's skills directory
(e.g. `.claude/skills/`). An agent will load it when writing EvaLuate scripts and immediately
know the frontmatter contract, the sandbox rules, the lifecycle hooks, and where to look up
exact API names (it points back at `spec/`).

**Reading:** `spec/evaluate-api.md` is the quickest way to find a class, method, or hook.

## Regenerate for your exact versions

The spec is produced by EvaLuate at runtime from the **live engine** — so it is exact for
*your* Godot build (including GDExtension classes) and *your* EvaLuate version, and it
re-derives automatically when either changes. Nothing is hand-written or hard-coded.

```
godot --headless --path . -- --emit-api downloads/spec
```

Run that from a project that hosts an `EvaluateRuntime` node (any EvaLuate game does). Re-run
it after upgrading Godot or EvaLuate. The committed `spec/` here is generated against the
Godot version this release was built with; each tagged release also ships a fresh
`evaluate-downloads-<version>.zip` on GitHub.
